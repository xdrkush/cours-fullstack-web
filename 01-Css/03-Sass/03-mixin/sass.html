03 Les mixins

Comment sa marche ?

Alors une mixin est comme une classe css mais qui n'en ai pas une, je m'explique:

Cela ressemble plus à un composant deja pré-enregisterer afin de rendre notre css plus modulaire, homogène et armonieux, que l'on va ensuite attribuer à une classe, id, une balise ...

Exemple on va enregistrer un comportement par default d'un boutton (mixin) 

        @mixin btn 

            border-radius: 25px

            padding: 10px

            min-width: 100px

            &:hover

                border: solid 2px $white

                box-shadow: 1px 1px 2px $secondary-color, 0 0 25px $secondary-color, 0 0 5px $secondary-color

Dans notre exemple on va faire utiliser ce comportement par une classe (.btn):

        .btn
            @include btn

Ce qui nous donne se boutton:

Ensuite on pourrais personalisez un peu plus notre boutton, voici 2 exemple: 

        .btn-success
            background: $green
            color: $secondary-color
            &:hover
                background: $green-dark
      
        .btn-canceled
            background: $red
            color: $secondary-color
            &:hover
                background: $red-dark

Dans la même catégory on pourrais aussi utiliser les extensions "extend" ?

Ceci est une classe que l'on va pouvoir étendre à une autre:

Par exemple: 

        .bg-green
            background-color: $green

        .btn-success
            @extend .bg-green
            color: $secondary-color
            &:hover
                background: $green-dark
